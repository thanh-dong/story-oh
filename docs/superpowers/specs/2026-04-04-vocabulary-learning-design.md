# Vocabulary Learning Feature — Design Spec

**Date:** 2026-04-04
**Status:** Approved

## Overview

A vocabulary learning system where parents request AI-generated weekly learning plans for their children. The app generates age-appropriate topics and word lists based on the child's profile (age, interests, native language, learning language). Kids learn through an interactive listen-and-repeat experience with personalized native-language voice prompts, followed by quick check quizzes for reinforcement.

## Core Flow

1. **Parent requests** a vocabulary plan (1-4 weeks) for a child, selecting the learning language
2. **AI generates** a draft plan — topics, words, prompt sentences, pacing — all adapted to the child's profile
3. **Parent reviews and approves** — credits are charged, TTS audio generation begins
4. **Plan goes active** — kid sees today's topic and word list in kid mode
5. **Kid learns** — taps words to hear personalized native-language prompts, completes quick checks
6. **Progress tracked** — per-word listening, quiz results, daily/weekly completion

|--------|------|-------------|
| id | uuid | Primary key |

| creditsCost | integer | Total credits charged |
| wordsTotal | integer | Total words in the plan (set on creation) |
| wordsAudioReady | integer | Count of words with TTS generated (default 0) |
| plan | jsonb | Full AI-generated plan structure (see below) |
| createdAt | timestamp | |
| updatedAt | timestamp | |

**Indexes:**
- `(childId, learningLanguage, status)` — enforce one active plan per child+language, fast lookup in kid mode
- **Unique constraint:** At most one plan with status `active` or `approved` per `(childId, learningLanguage)`. Parent must complete or cancel existing plan before creating a new one.

**Plan JSON structure:**

```json
{
  "weeks": [{
    "weekNumber": 1,
    "days": [{
      "day": 1,
      "topic": "Ocean Creatures",
      "isReview": false,
      "words": [
        { "word": "fish", "emoji": "🐟", "pronunciation": "/fɪʃ/", "promptSentence": "..." },
        { "word": "whale", "emoji": "🐋", "pronunciation": "/weɪl/", "promptSentence": "..." }
      ]
    }, {
      "day": 6,
      "topic": "Week Review: Animals",
      "isReview": true,
      "words": [
        { "word": "fish", "emoji": "🐟", "pronunciation": "/fɪʃ/", "promptSentence": "..." }
      ]
    }]
  }],
  "quizOptions": {
    "choiceCount": 3
  }
}
```

The AI generates the full word objects (with emoji, pronunciation in IPA, and promptSentence) as part of the plan output. Review day word lists are AI-curated during generation — the AI selects which words from the week to include.

## AI Generation

### Plan Generation

The AI receives the child's full profile and generates a structured weekly plan:

**Inputs:**
- Child's age (from DOB), name, interests
- Native language, learning language
- Number of weeks requested

**AI adapts by age:**
- Word complexity: "cat, dog, fish" for age 4-5 vs "dolphin, crocodile, habitat" for 10-12
- Words per topic: fewer for younger, more for older
- Topic depth: harder/broader topics span 2+ days, simple topics fit in 1 day
- Prompt sentence tone: simpler and more playful for young kids, slightly more instructional for older
- Quick check difficulty: 2 choices for young, 3-4 for older

**Pacing rules:**
- Monday through Friday: learning days with new topics
- Harder topics can span multiple days
- Saturday + Sunday: review/mix days combining the week's words
- Topics connect to the child's interests when possible

**Output:** Strict JSON schema matching the plan structure above, including all word details (emoji, pronunciation, promptSentence).

### Prompt Sentence Generation

The AI generates varied, natural native-language prompts for each word as part of the plan JSON. Requirements:
- Personalized with the child's name
- Warm, encouraging, conversational tone — like a parent or tutor
- "Repeat after me" woven in naturally with different phrasings
- Ends with the word spoken clearly in the learning language
- No repetitive templates — each sentence feels unique

**Example variations (Vietnamese native, learning English):**
- "[Name] oi, con ca tieng Anh la gi con, con lap lai nha. — Fish"
- "[Name] a, con biet con voi noi tieng Anh la gi khong? — Elephant. Con noi thu nha!"
- "Ne [Name], cai nay la con buom ne. Tieng Anh goi la — Butterfly. Noi theo nha!"

### TTS Audio Generation

- Triggered after parent approves the plan (not during draft)
- OpenAI TTS-1 with voice matched to native language (e.g., Vietnamese-sounding voice for Vietnamese prompts)
- Each word's promptSentence rendered as a single audio clip (MP3)
- **Durable storage:** Vocabulary audio is paid content — stored with a TTL of `(weeksRequested × 7 + 14) days` (plan duration + 2 weeks buffer for re-listening). This overrides the default 7-day TTS cache TTL by passing a custom `ttlMs` to `setCache()`.
- Cached by content hash — identical sentences across plans share the same file
- **Completion tracking:** After each word's audio is generated, increment `vocabularyPlans.wordsAudioReady`. When `wordsAudioReady === wordsTotal`, transition plan from `approved` → `active`.
- **Serving audio:** The client calls the existing `/api/tts` route with the word's `promptSentence` and voice parameters. On cache hit, audio is returned instantly. The `audioUrl` field on `vocabularyWords` stores the cache key for direct lookup.

**Error handling:**
- If TTS fails for individual words: retry up to 3 times with exponential backoff
- If TTS fails permanently for some words: plan still transitions to `active` with partial audio. Words without audio show a "retry" button in kid mode that calls `/api/tts` on demand.
- If TTS fails for ALL words: plan transitions to `failed`. Parent sees an error with option to retry (no additional credit charge) or cancel (credits refunded).

## Credit System

**Cost formula:**

```
estimateVocabularyCost(weeksRequested, childAge) =
  baseCost + (wordsPerWeek × weeksRequested × costPerWord)

where:
  baseCost = 3                           // AI plan generation
  wordsPerWeek = childAge <= 6 ? 25 : childAge <= 9 ? 35 : 45
  costPerWord = 0.15                     // TTS generation
  result = Math.round(baseCost + wordsPerWeek * weeksRequested * costPerWord)
```

Examples:
- Age 5, 1 week: 3 + (25 × 1 × 0.15) = ~7 credits
- Age 8, 1 week: 3 + (35 × 1 × 0.15) = ~8 credits
- Age 11, 2 weeks: 3 + (45 × 2 × 0.15) = ~17 credits

Proportional to story costs (15-25 credits). A week of vocabulary costs roughly half a story.

**Credit flow:**
- Credits charged on plan approval (not on draft generation)
- If parent rejects and regenerates, the regeneration costs another credit charge (warned beforehand). Maximum 3 regenerations per plan — after that, the plan is auto-cancelled.
- Transaction logged as `vocabulary_plan` type in existing `creditTransactions` ledger
- One language per plan — multi-language learning requires separate plans
- **Refund on failure:** If plan transitions to `failed` and parent cancels, credits are refunded (logged as `vocabulary_refund` transaction).

## Day Mapping Logic

**Calendar-based with catch-up allowed:**

- Day 1 = Monday of `weekStartDate`. Days map to fixed calendar dates.
- If a plan is approved mid-week (e.g., Wednesday), days 1-2 are immediately available for catch-up — the kid can do them on Wednesday. The "current day" highlights today's calendar day, but past unlocked days remain accessible.
- A day is "unlocked" if its calendar date ≤ today AND the plan is `active`.
- Future days (calendar date > today) are locked — the kid cannot skip ahead.
- **Missed days:** No penalty. The kid can catch up on missed days at any time while the plan is active. Progress shows which days are incomplete.
- **Plan completion:** A plan transitions to `completed` when all days (including review days) have their quick checks done, OR when the plan period ends (last day's calendar date + 7 day grace period passes). Incomplete plans can still be accessed in read-only mode after the grace period.

## Parent Flow (Dashboard)

### Entry Point

New "Vocabulary" card on the child's dashboard page (`/dashboard/[childId]`), alongside existing reading stats.

### Create Plan

1. Parent taps "Create Vocabulary Plan"
2. Selects: learning language (from child's `learningLanguages`), number of weeks (1-4)
3. Sees credit cost estimate (deterministic formula based on age and weeks)
4. Confirms → AI generates draft → loading state
5. **Guard:** If an active/approved plan already exists for this child+language, show a message: "A plan is already active. Complete or cancel it first."

### Review & Approve

- Draft displayed as a week view — days as rows, each showing topic + word list
- All weeks visible for multi-week plans
- Actions: "Approve" (charges credits, triggers TTS) or "Regenerate" (costs credits, warned, max 3 times)
- On approval: status → `approved`, TTS generation starts in background
- Parent sees "Preparing audio..." state until plan transitions to `active`

### Cancel Plan

- Parent can cancel a plan at any status:
  - `draft` → deleted, no credits charged
  - `approved` / `active` → transitions to `cancelled`, no refund (audio already generated or in progress)
  - `failed` → transitions to `cancelled`, credits refunded

### Monitor Progress

Active plan on child's dashboard shows:
- Current day/topic highlighted
- Per-day status: not started / in progress / completed
- Weekly summary: X/Y words listened, quiz accuracy %
- Overall plan progress bar
- Past completed/cancelled plans in history section

### Plan Lifecycle

```
draft → approved → active → completed
  ↓        ↓         ↓
  ↓     failed → cancelled (refund)
  ↓        ↓
  deleted  cancelled
```

## Kid Mode UI

### Entry Point

On the kid's reading hub (`/dashboard/[childId]/read`), a "Today's Words" section appears showing today's topic with emoji and word count.

### Route

`/dashboard/[childId]/read/vocabulary/[planId]` — the vocabulary learning screen for a specific plan.

### Vocabulary Screen — Desktop/Tablet (Split View)

**Left panel (~30%):**
- Vertical word list for today's topic
- Each word shows emoji + word text
- Listened words get a visual indicator (checkmark/star)
- Current active word highlighted

**Right panel (~70%):**
- Large emoji/illustration
- Word in large text (learning language)
- Pronunciation guide (IPA)
- Big play button for TTS audio
- Unlimited replays
- After listening, word marked as "listened" in left list

### Vocabulary Screen — Mobile (Portrait)

- Full-width word list initially
- Tapping a word slides up a bottom sheet with the detail view
- Back/swipe down to return to list

### Quick Check

- Appears after all words in a topic are listened to
- Multiple choice: show emoji, pick correct word from 2-4 options (age-dependent, set in `plan.quizOptions.choiceCount`)
- Quiz questions are constructed dynamically at quiz time by shuffling the day's words as options
- Correct → celebration animation (confetti/stars)
- Wrong → gentle "Try again!" with audio replay, one more chance (max 2 attempts per question)
- `quizCorrect` records final result, `quizAttempts` records total tries
- No score pressure — playful, not a test
- Completing quick check marks the day's topic as done

### Review Days (Weekends)

- Same UI, word list is AI-curated from the week (stored in plan JSON)
- Quick check mixes words from all week's topics
- Completing review marks the week as done

### Navigation

- Kid can revisit unlocked days' topics (re-listen)
- Current day is the default view
- Future days (calendar date > today) are locked
- "Hold to exit" button (existing pattern) to return to reading hub

### Degraded States

- **Word with no audio** (TTS failed): Show a retry button that calls `/api/tts` on demand. Word is still visible and learnable.
- **Offline/network error:** Audio that was previously played is browser-cached. New audio requires network.

## API Routes

| Route | Method | Purpose |
|-------|--------|---------|
| `/api/vocabulary/plans` | POST | Create draft plan (AI generation) |
| `/api/vocabulary/plans` | GET | List plans for a child (`?childId=...`) |
| `/api/vocabulary/plans/[planId]` | GET | Get plan details + words |
| `/api/vocabulary/plans/[planId]/approve` | POST | Approve, charge credits, trigger TTS |
| `/api/vocabulary/plans/[planId]/regenerate` | POST | Regenerate draft (charges credits, max 3) |
| `/api/vocabulary/plans/[planId]/cancel` | POST | Cancel plan (refund if `failed`) |
| `/api/vocabulary/progress` | POST | Record word listened / quiz result |
| `/api/vocabulary/progress/[planId]` | GET | Get progress for a plan |

TTS audio is served through the existing `/api/tts` route — vocabulary uses the same endpoint with cache-hit optimization. No separate vocabulary TTS route needed.

## Architecture

- **TTS batch generation** runs as background process after approval. Increments `wordsAudioReady` per word. Plan moves `approved` → `active` once `wordsAudioReady === wordsTotal` (or partial-ready with failures).
- **Progress writes** are simple POSTs — kid taps word, client fires request.
- **Child ownership verification** reuses existing `verifyChildOwnership()` — every endpoint checks parent owns the child.
- **Credit flow** follows existing `creditTransactions` ledger — deduct on approve, log as `vocabulary_plan`. Refund on failure+cancel as `vocabulary_refund`.
- **Audio caching** by content hash of prompt sentence, with extended TTL for vocabulary (plan duration + 2 weeks).

### New Modules

- `lib/vocabulary-generation.ts` — AI prompt construction & response parsing
- `lib/vocabulary-credits.ts` — cost estimation with deterministic formula

## Non-Goals (v1)

- Speech recognition for the repeat step
- Typing/spelling exercises
- Multi-language within a single plan
- Spaced repetition algorithm
- Word bank curation layer
- Gamification beyond quick checks (streaks, leaderboards, etc.)
- Offline-first with service worker prefetching
- Parent editing individual words in a plan
